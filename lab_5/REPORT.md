## Формулировка задания
Реализовать программу на C для работы с графом и алгоритмом на графах.

Для варианта `N = 5`:
```text
(N % 3) + 1 = (5 % 3) + 1 = 3
(N % 5) + 1 = (5 % 5) + 1 = 1
```

Поэтому необходимо реализовать:
1. представление графа списком рёбер
2. алгоритм поиска кратчайших путей Дейкстры

Дополнительные требования:
1. возможность запуска программы из CLI
2. чтение графа из файла
3. выгрузка графа на диск
4. выгрузка результата работы алгоритма на диск
5. unit-тесты через GTest
6. benchmark решения с указанием платформы

## Представление графа
Граф хранится списком рёбер. Каждое ребро содержит:
```text
from   вершина, из которой выходит ребро
to     вершина, в которую входит ребро
weight вес ребра
```

Основные структуры:
```c
typedef struct
{
    int from;
    int to;
    int weight;
} Edge;

typedef struct
{
    int vertices_count;
    int edges_count;
    int edges_capacity;
    Edge *edges;
} Graph;
```

Вершины нумеруются с `0` до `vertices_count - 1`. Веса рёбер должны быть неотрицательными, так как алгоритм Дейкстры не работает с отрицательными весами.

## Алгоритм Дейкстры
Алгоритм Дейкстры находит кратчайшие расстояния от одной стартовой вершины до всех остальных вершин графа.

Идея алгоритма:
1. всем вершинам присваивается расстояние `INF_DISTANCE`
2. стартовой вершине присваивается расстояние `0`
3. на каждом шаге выбирается необработанная вершина с минимальным текущим расстоянием
4. для всех исходящих из неё рёбер выполняется релаксация
5. если найден путь короче, расстояние до соседней вершины обновляется

Релаксация ребра:
```text
если distance[current] + edge.weight < distance[edge.to],
то distance[edge.to] = distance[current] + edge.weight
```

Так как граф представлен списком рёбер, для поиска исходящих рёбер текущей вершины выполняется проход по всему массиву рёбер и выбираются рёбра с `edge.from == current`.

## Формат входного файла
Первая строка:
```text
<количество_вершин> <количество_рёбер>
```

Далее каждое ребро:
```text
<откуда> <куда> <вес>
```

Пример:
```text
5 7
0 1 10
0 2 3
1 2 1
1 3 2
2 1 4
2 3 8
3 4 7
```

## Как запустить
Сборка программы:
```bash
make all
```

Запуск программы с графом из `input.txt`:
```bash
make run
```

Команда `make run` запускает:
```bash
./lab5 input.txt 0 graph_output.txt result_output.txt
```

Аргументы:
```text
input.txt          входной файл с графом
0                  стартовая вершина
graph_output.txt   файл для выгрузки графа
result_output.txt  файл для выгрузки результата
```

Интерактивный режим:
```bash
make run_cli
```

Очистка:
```bash
make clean
```

## Benchmark
Benchmark написан через Google Benchmark.

Сборка benchmark:
```bash
make bench
```

Запуск benchmark:
```bash
make run_bench
```

Результат сохраняется в файл:
```text
benchmark_result.json
```

Benchmark запускает алгоритм Дейкстры на графах разного размера:
```text
BM_Dijkstra/50/4
BM_Dijkstra/100/6
BM_Dijkstra/250/8
BM_Dijkstra/500/10
```

Первое число означает количество вершин. Второе число означает количество дополнительных рёбер из каждой вершины, кроме базовой цепочки рёбер.

Результат запуска от 18.05.2026:
```text
Benchmark                   Time             CPU   Iterations
BM_Dijkstra/50/4         22.6 us         22.5 us        30995
BM_Dijkstra/100/6         109 us          109 us         6392
BM_Dijkstra/250/8         800 us          799 us          875
BM_Dijkstra/500/10       3858 us         3796 us          185
```

Платформа:
```text
CPU: Apple M2
RAM: 8 GB
```

Предупреждения Google Benchmark на macOS:
```text
Unable to determine clock rate from sysctl
Failed to set thread affinity
```

не влияют на сами измерения времени.
